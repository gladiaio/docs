The sentiment and emotion model analyzes the transcript, detecting the general sentiment conveyed in each sentence as well as emotions that are being expressed.


## Usage

<Tabs>

<Tab title='Asynchronous STT'>

### Configuration

Enable sentiment and emotion analysis by setting the `sentiment_analysis` parameter to true in your transcription request.

```json
{
  "audio_url": "<your audio url>"
  "sentiment_analysis": true
}
```

### Results

Your transcription result will contain a `sentiment_analysis` key which will contain an array of all sentences in the audio, and for each one the conveyed sentiment and expressed emotion.

<Tip>
When `diarization` is enabled, the sentiment analysis output will contain the `speaker` as well, allowing you to analyze each speaker sperately.
</Tip>

```json
{
  "transcription":{...},
  "sentiment_analysis": {
    "success": true,
    "is_empty": false,
    "results": [
      {
        "text": "Jonathan, it says you are trained in technology.",
        "sentiment": "neutral",
        "emotion": "neutral",
        "start": 0.45158000000000004,
        "end": 2.364,
        "channel": 0,
        "speaker": 0,
      },
      {
        "text": "That's very good.",
        "sentiment": "positive",
        "emotion": "positive_surprise",
        "start": 2.54438,
        "end": 3.5432300000000003,
        "channel": 0,
        "speaker": 0,
      }
    ],
    "exec_time": 1.127103805541992,
    "error": null
  }
}
```

</Tab>

<Tab title='Real-Time STT'>

### Configuration

The sentiment analysis configuration is set within the `realtime_processing` object.

```json
{
  "realtime_processing": {
    "sentiment_analysis": true
  }
}
```

<Note>
Ensure that `messages_config.receive_realtime_processing_events` is set to `true` during initialization to receive sentiment analysis messages via the WebSocket.
</Note>

### Results

You can identify relevant messages by filtering on the `type` key by **sentiment_analysis**. Each message will include the analyzed utterance and a `results` key containing the detected sentiment and emotion.

```json
{
    "type": "sentiment_analysis",
    "session_id": "6054ff67-9eec-44bc-ac42-23b784cec41f",
    "created_at": "2024-12-26T14:20:55.027Z",
    "data": {
        "utterance_id": "1",
        "utterance": {
            "text": " And today I have a really good news to tell you.",
            "start": 3.2960000000000003,
            "end": 6.4719999999999995,
            "language": "en",
            "confidence": 1,
            "channel": 0,
            "words": [
                {
                ...
                }
            ]
        },
        "results": [
            {
                "text": "And today I have a really good news to tell you.",
                "sentiment": "positive",
                "emotion": "elation",
                "start": 3.296,
                "end": 6.4719999999999995,
                "speaker": 0,
                "channel": 0
            }
        ]
    },
    "error": null
}
```
