The summarization model generates a summary of your audio file. You can customize the result to match your preferences:
- general: a regular summary of the transcription. This is the default configuration.
- concise: a shorter summary for quick overview.
- bullet_points: retrieve the key points in a list.

## Usage

<Tabs>

<Tab title='Asynchronous STT'>

### Configuration

Enable summarization by setting the `summarization` parameter to true in your transcription request. API reference is available [here](https://docs.gladia.io/api-reference/v2/pre-recorded/init#body-summarization).

```json
{
  "audio_url": "<your audio url>"
  "summarization": true,
  "summarization_config": {
      "type": "general"
    }
}
```

### Results

Your transcription result will have a `summarization` key containing your audio file's summary.

```json
{
  "transcription": {...},
  "summarization": {
    "success": true,
    "is_empty": false,
    "results": "This transcription suggests that...",
    "exec_time": 1.5126123428344727,
    "error": null
  }
}
```

</Tab>

<Tab title='Real-Time STT'>

### Configuration

The summarization configuration is set within the `post_processing` object. API reference is available [here](https://docs.gladia.io/api-reference/v2/live/init#body-post-processing).

```json
{
  "post_processing": {
    "summarization": true,
    "summarization_config": {
      "type": "general"
    }
  }
}
```

<Note>
Ensure that `messages_config.receive_post_processing_events` is set to true during initialization to receive summarization messages via the WebSocket.
</Note>

### Results

You can identify relevant messages by filtering on the `type` key by **post_summarization**.

```json
{
  "type": "post_summarization",
  "session_id": "242ac259-91b8-4e28-8bf6-761ab06eafc5",
  "created_at": "2025-01-02T11:19:04.399Z",
  "data": {
    "results": "Certains clients signalent un manque de réactivité du support client pour les demandes de prise en charge des véhicules. Un meeting est prévu la semaine prochaine pour résoudre ce problème en révisant les processus internes."
  },
  "error": null
},
```

</Tab>
